# 终端显示与滚动修复计划

## 问题判断

是的，当前终端显示逻辑有问题：你看到“只显示 3 行/旧命令像被删除”不是 Git 行为，而是前端终端渲染与行管理策略造成的视觉丢失。

核心风险点在 `[/Users/ahs/Project/Web/git-learn/src/components/TerminalPanel.tsx](/Users/ahs/Project/Web/git-learn/src/components/TerminalPanel.tsx)`：

- 使用 ANSI 清行重绘（`\r\x1b[2K`）在异步输出时容易覆盖可见行。
- prompt 行和历史行混在同一个“重绘循环”里，存在竞态。
- 当前容器高度固定较小，若视窗行高计算不稳定，会放大“像只剩几行”的体感。

## 改造目标

- 终端历史仅追加，绝不主动清除历史行。
- 输入后立即换到下一行 prompt，旧命令永久保留在滚动历史中。
- terminal 区域可滚动，历史可向上翻阅。
- 保持已有焦点与行内编辑能力（左右移动、插入、回车后继续输入）。

## 实施步骤

1. 分离“历史输出层”与“输入行层”写入逻辑
  - 文件： [`/Users/ahs/Project/Web/git-learn/src/components/TerminalPanel.tsx`](/Users/ahs/Project/Web/git-learn/src/components/TerminalPanel.tsx) 
  - 历史输出只用 `writeln` 追加，不再用清行控制符覆盖历史。
  - 输入行只在当前光标行局部重绘，不影响已提交历史。
2. 去掉导致覆盖的清行路径
  - 文件： [`/Users/ahs/Project/Web/git-learn/src/components/TerminalPanel.tsx`](/Users/ahs/Project/Web/git-learn/src/components/TerminalPanel.tsx) 
  - 清理 `\r\x1b[2K` 在历史回放流程中的使用，仅保留最小必要光标定位。
3. 稳定行追加顺序
  - 文件： [`/Users/ahs/Project/Web/git-learn/src/App.tsx`](/Users/ahs/Project/Web/git-learn/src/App.tsx) 
  - 保证提交后按固定顺序写入：
    - `git-learn$ <command>`
    - `result lines...`
    - `git-learn$` （新输入行）
  - 统一由单一路径维护 `terminalHistory`，避免多处并发写入导致行缺失。
4. 强化滚动体验
  - 文件： [`/Users/ahs/Project/Web/git-learn/src/App.css`](/Users/ahs/Project/Web/git-learn/src/App.css) 
  - 确保 `.terminal-panel` 与 xterm 视口在固定高度内可滚动。
  - 调整行高与容器 padding，避免可见行数异常偏少。
5. 回归验证（Playwright CLI）
  - 连续执行 6~10 条命令，确认历史累计不丢行。
  - 验证能向上滚动查看早期命令。
  - 检查回车后无需点击可继续输入。
