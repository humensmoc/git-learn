# 夜间模式切换方案

## 目标

- 在顶栏「音效」按钮旁边增加主题切换按钮（如 `夜间模式: 开/关`）。
- 一键切换到暗色调，降低当前 LGB 白天模式的白底刺眼感。
- 刷新页面后记住用户选择。

## 实现策略

采用 **CSS 变量 + `data-theme` 属性**，避免维护两套完整样式文件。

```mermaid
flowchart LR
  topbarBtn[顶栏夜间模式按钮] --> appState[App theme state]
  appState --> htmlAttr["document.documentElement data-theme"]
  appState --> localStorage[localStorage git-learn-theme]
  htmlAttr --> cssVars[index.css 暗色 token]
  cssVars --> appCss[App.css 各面板样式]
  appState --> terminal[TerminalPanel xterm theme]
  appState --> graph[GitGraph 空状态配色]
```

## 1. 主题状态与按钮

修改 [`src/App.tsx`](src/App.tsx)：

- 新增 `const [darkMode, setDarkMode] = useState(false)`。
- 启动时从 `localStorage.getItem("git-learn-theme")` 读取（`"dark"` / `"light"`）。
- `useEffect` 同步到 `document.documentElement.dataset.theme = darkMode ? "dark" : "light"`。
- 在音效按钮旁新增：

```tsx
<button type="button" onClick={() => setDarkMode((v) => !v)}>
  夜间模式: {darkMode ? "开" : "关"}
</button>
```

## 2. 暗色主题 token

修改 [`src/index.css`](src/index.css)：

- 保留现有 `:root` 作为 **light** 默认主题。
- 新增 `[data-theme="dark"]` 覆盖核心变量：

| 变量 | 日间（现有） | 夜间（建议） |
|------|-------------|-------------|
| `--canvas-blue` | `#8fd3f4` | `#0f1a2e` |
| `--window-chrome` | `#e8e8e8` | `#2a3544` |
| `--bg-elevated` | `#ffffff` | `#1a2433` |
| `--bg-soft` | `#fafbff` | `#141c28` |
| `--border` | `#b8c6d6` | `#334155` |
| `--text` | `#162030` | `#e2e8f0` |
| `--text-soft` | `#44576f` | `#94a3b8` |
| `--graph-node` | `#ffffff` | `#e2e8f0` |
| `--graph-edge` | `#1a1a1a` | `#cbd5e1` |
| `--lime-bar` | `#c8e600` | `#8fa800`（略降饱和） |
| `--goal-pink` | `#ff6eb4` | `#b84a7a` |

- `color-scheme` 在 dark 下设为 `dark`。

## 3. 将 App.css 关键硬编码改为变量

修改 [`src/App.css`](src/App.css)（优先改高频区域，不必一次改完 68 处）：

- **顶栏** `.topbar`：背景/边框/阴影改用 `var(--bg-elevated)`、`var(--border)`。
- **按钮** `button`：背景/文字/边框改用主题变量。
- **窗口外壳** `.window-chrome`、`.window-titlebar`、`.window-body`：改用 `--bg-elevated`、`--window-chrome`、`--border`。
- **课程区** `.lesson-panel`、`.lesson-feedback`、`.lesson-panel code`：文字与底色跟随变量。
- **目标浮窗** `.goal-panel`、`.goal-panel-header`、`.goal-panel-body`：使用 `--goal-pink` 与暗色文字变量。
- **Graph 区域** `.graph-scroll`：背景改为 `var(--bg-soft)` 半透明。
- **风险徽章** `.risk-*`：在 `[data-theme="dark"]` 下微调对比度（避免暗底上文字发灰）。

对暂时未变量化的零星 hex，可在 `[data-theme="dark"]` 下用局部 override 兜底（如 `.topbar`、`.lesson-actionbar-buttons button`）。

## 4. 终端与 Graph 适配

### TerminalPanel

修改 [`src/components/TerminalPanel.tsx`](src/components/TerminalPanel.tsx)：

- 新增可选 prop：`darkMode?: boolean`（由 App 传入）。
- 在 `useEffect` 中监听 `darkMode`，调用 `term.options.theme = {...}` 更新 xterm 配色：
  - 夜间：背景 `#141c28`，前景 `#e2e8f0`，光标 `#c8e600`（保持 LGB 风格）。
  - 日间：维持现有 `#3a3a3a` 终端区（终端窗口本身已是深色，变化较小）。

### GitGraph

修改 [`src/components/GitGraph.tsx`](src/components/GitGraph.tsx)：

- 空状态卡片与文字颜色改为 CSS 变量（如 `--graph-empty-bg`、`--graph-empty-text`），在 `index.css` 的 dark token 中定义。
- 提交节点描边/文字在暗色下改用 `var(--text)`，避免黑字在暗底不可读。

## 5. 持久化与默认行为

- `localStorage` 键名：`git-learn-theme`。
- 默认值：**light**（保持现有 LGB 白天视觉为默认）。
- 切换时立即生效，无需刷新。

## 6. 验证

- 目视：点击「夜间模式」后，画布、顶栏、课程窗、文件窗、目标窗明显变暗，不再刺眼。
- 功能：刷新后主题保持；终端输入与 Graph 在两种主题下均可读。
- 构建：`npm run build` 通过。

## 不涉及

- 不新增独立主题设置页。
- 不跟随系统 `prefers-color-scheme` 自动切换（仅手动按钮控制；后续可加）。
