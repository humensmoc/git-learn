# 终端历史全程保留计划

## 目标

- 命令行历史在整次学习过程中持续保留。
- 完成步骤、切换 World 都不清空 terminal 内容。
- 历史只追加，必要时加“切换关卡”提示行便于阅读。

## 问题定位

当前在 [`/Users/ahs/Project/Web/git-learn/src/App.tsx`](/Users/ahs/Project/Web/git-learn/src/App.tsx) 的 `worldIndex` 切换逻辑中存在 `setTerminalHistory([])`，这会在进入新 World 时清空终端历史。

## 实施步骤

1. 移除切换 World 时的清空逻辑
   - 文件：[`/Users/ahs/Project/Web/git-learn/src/App.tsx`](/Users/ahs/Project/Web/git-learn/src/App.tsx)
   - 删除 `setTerminalHistory([])`。

2. 将“进入关卡”改为追加提示而不是重置
   - 文件：[`/Users/ahs/Project/Web/git-learn/src/App.tsx`](/Users/ahs/Project/Web/git-learn/src/App.tsx)
   - 在 `worldIndex` 变化时追加一行分隔提示，例如：
     - `----- 进入 World X -----`
   - 不影响已有命令输出历史。

3. 保持终端只追加写入策略
   - 文件：[`/Users/ahs/Project/Web/git-learn/src/components/TerminalPanel.tsx`](/Users/ahs/Project/Web/git-learn/src/components/TerminalPanel.tsx)
   - 维持当前“只追加 + 滚动查看”逻辑，不引入任何清空分支。

4. 回归验证
   - 场景 A：执行数条命令后切换 World，确认旧命令仍在。
   - 场景 B：完成步骤后历史不丢失，仅继续追加。
   - 场景 C：滚动能查看早期命令。
