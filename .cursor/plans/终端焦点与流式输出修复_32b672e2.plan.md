# 终端焦点与连续输出修复计划

## 现状与根因

基于 [`/Users/ahs/Project/Web/git-learn/src/components/TerminalPanel.tsx`](/Users/ahs/Project/Web/git-learn/src/components/TerminalPanel.tsx) 的当前实现：

- 回车后虽然重绘了提示符，但没有显式把焦点锁回 xterm 的输入区域（`textarea`），所以浏览器偶发丢焦，需要再次点击才能继续输入。
- 历史写入通过 `history` 状态异步回流到终端，当前“先清当前行再写历史再重绘”的流程在节奏上会产生“像刷新一屏”的感受。

## 改造目标

- 按回车后立即进入下一行可输入状态（无需鼠标点击）。
- 终端历史保持真实 CLI 体验：命令与输出持续向下追加。
- 兼容现有行内编辑、补全、上下历史回放逻辑。

## 实施步骤

1. 增加稳定焦点控制
   - 文件：[`/Users/ahs/Project/Web/git-learn/src/components/TerminalPanel.tsx`](/Users/ahs/Project/Web/git-learn/src/components/TerminalPanel.tsx)
   - 抽出 `focusTerminal()`，在以下时机调用：
     - 终端初始化后
     - `Enter` 提交命令后立即
     - `history` effect 回写完成后
   - 同步处理 xterm 的 `blur` 场景（容器点击后强制回焦）。

2. 优化“历史追加”渲染策略
   - 文件：[`/Users/ahs/Project/Web/git-learn/src/components/TerminalPanel.tsx`](/Users/ahs/Project/Web/git-learn/src/components/TerminalPanel.tsx)
   - 保留当前输入缓冲区快照，写入历史时只追加新增行，不对整行做“清空后重建”式视觉跳变。
   - 统一通过 `appendOutputLines()` + `renderPromptLine()` 保证顺序：
     1) 追加历史
     2) 换行显示 prompt
     3) 光标定位到输入位

3. 消除回车与异步结果的竞态
   - 文件：[`/Users/ahs/Project/Web/git-learn/src/components/TerminalPanel.tsx`](/Users/ahs/Project/Web/git-learn/src/components/TerminalPanel.tsx)
   - 引入轻量“命令提交中”标记，避免在历史回流瞬间重复重绘输入行造成光标跳跃。
   - 确保 `printedRef` 与 `history` 同步仅处理新增片段，避免重复写入或覆盖错觉。

4. 保持滚动与视觉稳定
   - 文件：[`/Users/ahs/Project/Web/git-learn/src/components/TerminalPanel.tsx`](/Users/ahs/Project/Web/git-learn/src/components/TerminalPanel.tsx)
   - 输出后统一 `scrollToBottom()`，输入提示不触发额外跳屏。
   - 文件：[`/Users/ahs/Project/Web/git-learn/src/App.css`](/Users/ahs/Project/Web/git-learn/src/App.css)
   - 若有必要，仅微调 `.terminal-panel` 以避免外层容器抢焦点或遮挡输入层。

5. 回归验证（Playwright CLI）
   - 场景 A：输入 `git init` 回车，紧接着输入 `git status`（不点击鼠标）应可直接输入。
   - 场景 B：连续输入 `levels` / `hint` / `git status`，历史按时间向下累计。
   - 场景 C：输入长输出后滚动到底部，提示符始终在最新底部并可继续输入。
