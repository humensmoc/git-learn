# 模拟终端输入与滚动修复计划

## 问题定位

基于当前实现，问题集中在 `onData` 的输入模型和输出渲染策略：

- 在 [`/Users/ahs/Project/Web/git-learn/src/components/TerminalPanel.tsx`](/Users/ahs/Project/Web/git-learn/src/components/TerminalPanel.tsx) 中，输入状态只有 `lineRef` 字符串，没有“光标索引”，所以只能在末尾追加字符，无法行内移动和编辑。
- 该文件的历史输出渲染使用 `\r\x1b[2K` 清行重绘，导致你看到“新输出覆盖旧行”的体验。
- 终端滚动虽由 xterm 支持，但当前写入流程没有稳定区分“历史区”和“输入区”，导致光标/历史行为不稳定。

## 目标行为

- 左右方向键可在当前输入行内移动光标。
- 在光标中间输入字符可插入，不是只能追加到末尾。
- Backspace 删除光标前字符（不是总删末尾）。
- 每次执行命令后，`命令 + 返回结果` 作为历史追加在下方。
- 终端区域可滚动，且新输出后自动滚到底部。

## 实施步骤

1. 重构输入状态为“文本 + 光标位置”
   - 在 [`/Users/ahs/Project/Web/git-learn/src/components/TerminalPanel.tsx`](/Users/ahs/Project/Web/git-learn/src/components/TerminalPanel.tsx) 增加：
     - `lineBufferRef`：当前输入内容
     - `cursorPosRef`：当前光标索引
   - 新增 `renderPromptLine()`：统一重绘 `PROMPT + lineBuffer`，并将终端光标移动到 `cursorPos`。

2. 完整处理键盘事件（行内编辑）
   - `ArrowLeft` / `ArrowRight`：仅移动 `cursorPos`，不改文本。
   - 普通字符：插入到 `cursorPos` 处，`cursorPos + 1`。
   - `Backspace`：删除 `cursorPos - 1` 字符并左移。
   - `Enter`：提交当前行到 `onCommand`，然后开启新提示行。
   - 保持 `ArrowUp/ArrowDown` 历史命令回放，并同步到 `lineBufferRef + cursorPosRef`。

3. 改造历史输出写入策略（只追加不覆盖）
   - 在 [`/Users/ahs/Project/Web/git-learn/src/components/TerminalPanel.tsx`](/Users/ahs/Project/Web/git-learn/src/components/TerminalPanel.tsx) 的 `history` effect 中：
     - 不再使用 `\r\x1b[2K` 清屏模式。
     - 先暂存当前输入行，换行输出新增历史，再重绘输入行。
   - 确保每条历史是独立行，旧内容永不覆盖。

4. 滚动行为与容器稳定性
   - 在输出后调用 xterm 滚动到底部（例如 `scrollToBottom` 或等效控制）。
   - 保持 [`/Users/ahs/Project/Web/git-learn/src/App.css`](/Users/ahs/Project/Web/git-learn/src/App.css) 中 `.terminal-panel` 的 `overflow: hidden`，由 xterm 自身滚动层负责滚动。

5. 联调与回归
   - 场景回归（手动 + Playwright CLI）：
     - 输入 `::` 后按左键，光标应回到中间位置。
     - 在中间插入字符，命令文本应正确更新。
     - 连续执行 `git status`、`hint`、`levels`，历史应逐条向下追加。
     - 长输出时终端能滚动，并在新输出后保持底部可见。
