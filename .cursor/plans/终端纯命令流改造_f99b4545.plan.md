# 终端纯命令流改造计划

## 目标

- 终端区域只保留“命令行本体”体验：`git-learn$`、输入命令、命令输出。
- 初始欢迎文案、模式提示、教学提示全部不出现在终端。
- `levels / hint / reset` 保留可用，但其教学反馈不在终端显示，改为上方 LessonPanel 呈现。
- 命令历史按真实终端样式逐行追加，不覆盖旧行。

## 实施步骤

1. 清理终端初始化输出
   - 文件：[ `/Users/ahs/Project/Web/git-learn/src/components/TerminalPanel.tsx` ](/Users/ahs/Project/Web/git-learn/src/components/TerminalPanel.tsx)
   - 移除初始化时的欢迎文案写入。
   - 首屏仅渲染一行 `git-learn$ ` 提示符。

2. 终端只显示命令与结果
   - 文件：[ `/Users/ahs/Project/Web/git-learn/src/components/TerminalPanel.tsx` ](/Users/ahs/Project/Web/git-learn/src/components/TerminalPanel.tsx)
   - 固化输出顺序：
     - 提交时写入 `> 用户命令`（或按现样式直接 `git-learn$ 命令`）
     - 紧接着写命令结果
     - 自动落到下一行 `git-learn$ `
   - 保持焦点锁定与滚动到底部。

3. 教学内置命令改为“面板反馈”
   - 文件：[ `/Users/ahs/Project/Web/git-learn/src/App.tsx` ](/Users/ahs/Project/Web/git-learn/src/App.tsx)
   - 保留 `levels`、`hint`、`reset` 分支逻辑。
   - 这些命令不再写入 `terminalHistory`，改为更新 LessonPanel 状态（例如“当前关卡列表”、“当前步骤提示”、“已重置”状态提示）。

4. LessonPanel 承担教学提示与内置命令反馈
   - 文件：[ `/Users/ahs/Project/Web/git-learn/src/components/LessonPanel.tsx` ](/Users/ahs/Project/Web/git-learn/src/components/LessonPanel.tsx)
   - 保持并强化当前步骤说明、命令提示（`commandHint`）与进度。
   - 增加一个轻量“提示反馈区”，用于显示 `levels/hint/reset` 的结果文本。

5. 回归验证（Playwright CLI）
   - 场景 A：刷新后终端首屏只有 prompt，不含欢迎/模式文案。
   - 场景 B：输入 `git init`、`git status`，历史逐行追加。
   - 场景 C：输入 `hint` 或 `levels`，验证终端不出现教学提示文本，但 LessonPanel 提示反馈区有更新。
