# 终端快捷坞增强计划

## 目标
让快捷按钮区更完整、更好学：
- 新增 `diff` 快捷按钮及常用参数子选项。
- 扩展现有按钮的“空格+减号参数”子功能。
- 为主按钮与子按钮增加 hover 提示框，内容包含“做什么 / 会执行什么 / 常见场景”，鼠标移开即隐藏。

## 变更范围
- 快捷命令数据与命令拼装：[`src/terminal/shortcuts.ts`](src/terminal/shortcuts.ts)
- 快捷坞交互与 tooltip 渲染：[`src/components/ShortcutDock.tsx`](src/components/ShortcutDock.tsx)
- 快捷坞与提示框样式：[`src/App.css`](src/App.css)

## 具体实施
1. 在 `shortcuts` 数据层增加描述元数据
- 给 `GitShortcut` 与 `ShortcutOption` 增加 tooltip 所需字段（如 `purpose`、`effect`、`scenario`）。
- 保持 `buildShortcutCommand()` 兼容现有逻辑，新增字段不影响命令拼接。

2. 新增 `diff` 主按钮与子选项
- 新增主按钮：`git diff`。
- 按你选择加入子选项：`--staged`、`--name-only`、`--stat`、`--cached`。
- 保持点击行为一致：主按钮可直接执行基础命令，子按钮执行带参数命令。

3. 扩展其他按钮的常用减号参数子功能（在不引入高风险误导前提下）
- 重点补充常用参数型子选项，例如：
  - `status`: `-s`、`-sb`
  - `log`: `--oneline`（已存在）+ `--graph`、`--decorate`
  - `push`: `-u origin main`（已存在）+ `--force-with-lease`
  - `remote`: 新增 `set-url` 快捷输入
  - `restore`: 增加 `--source` 形式的参数输入（保守提供）
- 原则：优先“教学场景常见 + 当前引擎可稳定反馈”的参数，避免堆太冷门或易误解的 flag。

4. 在 `ShortcutDock` 增加 hover 提示框交互
- 目标元素：主按钮 `pill-main`、子按钮 `pill-sub`。
- 行为：`mouseenter` 显示、`mouseleave` 隐藏；只显示一个提示框。
- 内容结构（中等密度）：
  - 做什么（purpose）
  - 会执行什么（effect，展示对应命令模板）
  - 常见场景（scenario）
- 定位策略：提示框跟随被 hover 按钮锚点显示（优先按钮下方，空间不足时自动上翻）。

5. 样式与可读性
- 新增 tooltip 样式（小卡片、边框、阴影、较小字号），与深色终端风格一致。
- 保证层级（z-index）高于快捷坞但不遮挡输入弹窗。

6. 回归验证
- 手动验证：
  - `diff` 新按钮和子选项均能触发正确命令。
  - 扩展后的其他按钮子功能可正常触发。
  - hover 出现提示，移开立即消失。
  - 展开/收起子按钮时 tooltip 不抖动、不残留。
- 运行构建检查（不改业务逻辑前提下确保 TS/CSS 无错误）。

## 兼容性与风险控制
- 不改 lesson validator，不改课程进度逻辑，只增强快捷输入层。
- 参数扩展采用“保守默认”，对高风险命令保留提示语（例如 `--force-with-lease` 标注协作场景注意点）。
