# Git Learn 使用指南

本文说明如何**在本地运行本项目**，以及打开网站后**如何学习 Git**。若你只想学 Git、不打算改代码，可直接跳到 [二、网站怎么用](#二网站怎么用)。

---

## 一、本地运行项目

### 1. 环境要求

| 依赖 | 版本 |
|------|------|
| Node.js | 18 或更高 |
| npm | 随 Node 安装即可 |

检查版本：

```bash
node -v
npm -v
```

### 2. 首次安装

在项目根目录执行：

```bash
# 进入项目目录（若尚未进入）
cd git-learn

# 安装依赖（只需第一次，或 package.json 变更后）
npm install
```

### 3. 启动开发服务器

```bash
npm run dev
```

终端会输出本地地址，一般为：

```
http://localhost:5173
```

用浏览器打开该地址即可。修改源码后页面会自动热更新，无需手动刷新。

### 4. 常用 npm 命令

| 命令 | 用途 |
|------|------|
| `npm run dev` | 启动开发服务器（日常开发、学习用这个） |
| `npm run build` | 类型检查 + 打包生产版本到 `dist/` |
| `npm run preview` | 在本地预览生产构建（需先 `npm run build`） |
| `npm run lint` | 运行 ESLint 检查代码规范 |

**典型工作流：**

```bash
npm install      # 第一次
npm run dev      # 日常开发 / 学习
npm run build    # 准备部署前
npm run preview  # 验证打包结果
```

### 5. 部署说明（可选）

本项目是纯前端静态站点，构建产物在 `dist/` 目录。可部署到任意静态托管（GitHub Pages、Vercel、Netlify 等）：

```bash
npm run build
# 将 dist/ 目录上传或交给 CI 发布
```

## 二、网站怎么用

### 1. 界面布局

打开网站后，主要区域如下：

```
┌─────────────────────────────────────────────────────────────┐
│  顶栏：上一关 / 下一关 / 音效 / 夜间模式                      │
├──────────────────────┬──────────────────────────────────────┤
│  左栏（约 35%）       │  右栏（约 65%）                       │
│  · 学习面板（关卡说明）│  · Git 提交图谱（Graph View）         │
│  · 终端（输入命令）   │  · 目标浮层（当前步骤，可隐藏）        │
│  · 文件状态           │                                      │
└──────────────────────┴──────────────────────────────────────┘
```

- **学习面板**：当前关卡名称、步骤说明、示例命令、进度与反馈。
- **终端**：像真实 Shell 一样输入 Git 命令；支持上下键历史、Tab 补全。
- **文件状态**：工作区与暂存区文件列表。
- **Graph View**：提交、分支、标签、远程的可视化图谱，命令执行后会动画更新。
- **目标浮层**：右侧图谱上的当前步骤摘要；可通过学习面板的「隐藏目标 / 显示目标」切换。

左右分栏宽度可拖拽调整。

### 2. 学习流程

1. 阅读左上方**学习面板**中的步骤说明与 `commandHint`（建议输入的命令）。
2. 在**终端**中输入对应 Git 命令，按 Enter 执行。
3. 系统会校验**命令内容**和**仓库状态**；通过后自动进入下一步，学习面板会显示「当前步骤已完成，继续下一步。」
4. 若命令执行了但未满足步骤条件，终端仍会显示 Git 输出，需根据提示继续尝试。
5. 完成当前关卡所有步骤后，点击顶栏或学习面板中的**下一关**进入下一 World。

**进度会自动保存**到浏览器 `localStorage`（键名 `git-learn-progress`）。下次打开同一浏览器会从上次的关卡与步骤继续。清浏览器站点数据会重置进度。

### 3. 终端内置辅助命令

除 Git 命令外，终端还支持三个**内置命令**（不参与 Git 引擎，仅用于学习辅助）：

| 命令 | 作用 |
|------|------|
| `hint` | 显示当前步骤的命令提示；若有安全说明也会一并显示 |
| `levels` | 列出全部 13 个关卡的名称 |
| `reset` | 重置当前关卡的仓库状态，并从第 1 步重新开始 |

与学习面板上的「提示」按钮效果相同；`reset` 适合练错后从头再来。

### 4. 终端操作技巧

- **Tab**：命令补全（内置命令 + 当前引擎支持的 Git 子命令）。
- **↑ / ↓**：浏览本关已输入过的命令历史。
- 终端输出会保留在本关会话中；切换关卡时会在历史中插入分隔线（如 `----- 进入 World X -----`）。

### 5. 顶栏设置

| 按钮 | 说明 |
|------|------|
| 上一关 / 下一关 | 切换 World；换关会重置该关仓库并回到第 1 步 |
| 音效 | 输入命令时的短促按键音，开/关 |
| 夜间模式 | 明暗主题切换，偏好写入 `localStorage`（键名 `git-learn-theme`） |

### 6. 关卡一览

共 **13 个 World**，由浅入深：

| # | 主题 | 引擎 |
|---|------|------|
| 1 | 初始化仓库 | Sim（模拟） |
| 2 | 连接 GitHub | Sim |
| 3 | 克隆与同步 | Sim |
| 4 | 状态与诊断 | Sim |
| 5 | 提交流程 | Sim |
| 6 | 分支操作 | Sim |
| 7 | 合并与变基 | Sim |
| 8 | 暂存现场（stash） | Sim |
| 9 | 撤销基础 | Sim |
| 10 | 风险与回滚 | Sim |
| 11 | 忽略规则（`.gitignore`） | Sim |
| 12 | 协作收尾 | Sim |
| 13 | 真实沙盒收官 | **Real（真实 Git）** |

- **Sim 模式（第 1–12 关）**：轻量模拟引擎，响应快、无真实文件系统，适合安全练习。
- **Real 模式（第 13 关）**：基于 [isomorphic-git](https://isomorphic-git.org/)，在浏览器内运行真实 Git 仓库逻辑。

部分步骤带有**风险等级**（基础 / 进阶 / 危险）和**安全提示**（如已 push 的历史优先用 `revert` 而非 `reset --hard`），请留意学习面板与目标浮层中的说明。

逐步详情见 [docs/lesson/现有课程目录.md](./lesson/现有课程目录.md)；按实际使用场景重组的学习路径见 [docs/lesson/场景化课程规划.md](./lesson/场景化课程规划.md)。

## 四、常见问题

**Q：页面打不开 / 端口被占用？**  
Vite 若 5173 被占用会自动尝试其他端口，以终端实际输出的 URL 为准。

**Q：命令输入了但步骤不过？**  
对照学习面板中的 `commandHint`；校验同时看**命令写法**和**仓库状态**（例如是否已有提交、分支名是否正确）。可用 `hint` 再看提示，或用 `reset` 重来。

**Q：换关后之前的终端输出还在？**  
换关不会清空整段终端历史，但会插入关卡分隔线并重置仓库；每关从第 1 步开始。

**Q：进度丢了？**  
进度存在浏览器本地。换浏览器、无痕模式或清除站点数据会丢失进度。

**Q：和真实 Git 完全一致吗？**  
Sim 模式为教学简化；Real 模式更接近真实 Git，但仍运行在浏览器沙盒中，与本地 CLI 在边界行为上可能有差异。以第 13 关体验「更接近真实」的行为即可。

## 五、快速对照：我只想……

| 目标 | 操作 |
|------|------|
| 在浏览器里学 Git | `npm run dev` → 打开 localhost → 按学习面板提示在终端输入命令 |
| 从某一关开始 | 顶栏或学习面板点「上一关 / 下一关」；或终端输入 `levels` 查看列表后切换 |
| 卡在某一步 | 终端输入 `hint`，或点学习面板「提示」 |
| 本关练乱了 | 终端输入 `reset` |
| 改关卡内容 | 编辑 `src/lessons/data.ts` |
| 打包上线 | `npm run build` → 部署 `dist/` |
