# 课程目录（场景化）

本文档是 [`src/lessons/data.ts`](../../src/lessons/data.ts) 的可读镜像。课程已从命令型 World 重构为场景型 S01–S15。**修改课程以代码为准**，本文档用于查阅。

- **统计**：15 个场景、60 个 Step
- **引擎**：S01–S14 为 Sim（模拟）；S15 为 Real（真实 Git）
- **如何使用网站**：见 [使用指南](../使用指南.md)
- **设计来源**：见 [场景化课程规划](./场景化课程规划.md)

---

## 类型说明

| 字段 | 含义 |
|------|------|
| `mode` | `sim` = 轻量模拟引擎；`real` = 浏览器内 isomorphic-git |
| `seedId` | 关卡初始仓库状态，定义见 [`src/engine/seed.ts`](../../src/engine/seed.ts) |
| `riskLevel` | `basic` 基础 / `advanced` 进阶 / `danger` 危险 |
| `riskNote` | 可选安全提示文案 |

## 场景详情

### S01 — 本地新项目
- **id**: `scenario01`
- **核心命令**: `git init`、`git add .`、`git commit -m`、`git log --oneline`、`git status`
- **步骤**: `s01-init`、`s01-add-all`、`s01-commit`、`s01-log-oneline`、`s01-status`

### S02 — 日常改动提交
- **id**: `scenario02`
- **核心命令**: `git add <file>`、`git add .`、`git commit -m`、`git commit -am`
- **步骤**: `s02-init`、`s02-add-file`、`s02-commit-m`、`s02-add-all`、`s02-commit-am`

### S03 — 诊断状态差异
- **id**: `scenario03`
- **核心命令**: `git status`、`git diff`、`git diff --staged`、`git remote -v`
- **步骤**: `s03-init`、`s03-status`、`s03-diff`、`s03-diff-staged`、`s03-remote-v`

### S04 — 第一次推 GitHub
- **id**: `scenario04`
- **核心命令**: `git branch -M main`、`git remote add origin`、`git push -u origin main`
- **步骤**: `s04-init`、`s04-branch-m`、`s04-remote-add`、`s04-push-u`

### S05 — 换电脑克隆同步
- **id**: `scenario05`
- **核心命令**: `git clone`、`git fetch`、`git pull`、`git pull --rebase`
- **步骤**: `s05-clone`、`s05-fetch`、`s05-pull`、`s05-pull-rebase`

### S06 — 更换远程地址
- **id**: `scenario06`
- **核心命令**: `git remote -v`、`git remote set-url origin`
- **步骤**: `s06-remote-v`、`s06-remote-set-url`

### S07 — 开分支做功能
- **id**: `scenario07`
- **核心命令**: `git branch`、`git branch <name>`、`git switch -c`、`git checkout -b`
- **步骤**: `s07-init`、`s07-branch-list`、`s07-branch-create`、`s07-switch-c`、`s07-checkout-b`

### S08 — 合并或变基
- **id**: `scenario08`
- **核心命令**: `git merge`、`git rebase`
- **步骤**: `s08-setup`、`s08-merge`、`s08-rebase`

### S09 — 拉队友最新代码
- **id**: `scenario09`
- **核心命令**: `git fetch origin`、`git pull`、`git pull --rebase`
- **步骤**: `s09-fetch-origin`、`s09-pull`、`s09-pull-rebase`

### S10 — PR 前整理历史
- **id**: `scenario10`
- **核心命令**: `git fetch origin`、`git rebase origin/main`、`git log origin/main..HEAD`
- **步骤**: `s10-fetch-origin`、`s10-rebase-origin-main`、`s10-log-range`

### S11 — 被打断暂存现场
- **id**: `scenario11`
- **核心命令**: `git stash`、`git stash list`、`git stash pop`
- **步骤**: `s11-init`、`s11-stash`、`s11-stash-list`、`s11-stash-pop`

### S12 — 撤销还没推送的改动
- **id**: `scenario12`
- **核心命令**: `git restore`、`git restore --staged`、`git reset HEAD`、`git reset --soft`
- **步骤**: `s12-init-commit`、`s12-restore-staged`、`s12-restore-file`、`s12-reset-head`、`s12-reset-soft`

### S13 — 已推送后的安全回滚
- **id**: `scenario13`
- **核心命令**: `git commit --amend`、`git revert`、`git reset --hard`
- **步骤**: `s13-setup`、`s13-amend`、`s13-revert`、`s13-reset-hard`

### S14 — 忽略不该提交的文件
- **id**: `scenario14`
- **核心命令**: `.gitignore`、`git rm -r --cached`
- **步骤**: `s14-init`、`s14-gitignore`、`s14-add-tracked`、`s14-rm-cached`

### S15 — 真实环境演练
- **id**: `scenario15`
- **核心命令**: `git init`、`git add demo.txt`、`git commit -m`、`git log`
- **步骤**: `s15-init`、`s15-add`、`s15-commit`、`s15-log`

## 与旧目录的关系

- 原 `World 1–13` 命令型目录已由 S01–S15 场景型目录替代。
- 代码层保留了既有 `validatorId`，因此校验逻辑无需重写，重点变为课程顺序与叙事场景。
- `levels` 命令会直接列出新的场景标题。
